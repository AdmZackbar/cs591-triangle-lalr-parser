CS 491/591 Programming Project 2

Spring 2020

Project 2 is required for CS 591 students, and is optional for CS 491 students.

Project Description

Rewrite the lexical and syntactic phases of the Triangle compiler using Lex and Yacc.  You will need to modify the Triangle grammar so that it is LALR 1 rather than LL 1, because Yacc uses shift-reduce parsing rather than recursive-descent.  Given a syntactically-correct Triangle input program, your new Triangle parser should write the grammar rules of a rightmost derivation in reverse order to an output file.  However, if the Triangle input program contains any lexical or syntax errors, then your Triangle parser should write one or more appropriate error messages to standard output.  It is not necessary to report other kinds of errors (such as type errors) or to generate TAM code.

Project Rules

Your project must compile and run on the cs-parallel.ua.edu server.

All the code that appears in your program must be your individual work with no assistance from others.  Do not copy code from other students, or from the web or any other source.  Also do not share your code with other students.  If your program violates these rules, you will not receive credit for the project, and the violation may also be reported as academic misconduct.

The project is officially due by April 17, but there is a grace period so that you may submit without late penalty until April 24.  Please send me an email message by April 17 to schedule an appointment for you to demonstrate your project during the last week of classes (April 20 to 24).  Bring your laptop to my office, and allow about 20 to 30 minutes for the demo.

By April 24, upload a zip file to Blackboard that contains all your source code and input/output files and compilation/usage instructions for each project.  Make sure the zip file contains the final version of your program that you want graded.
